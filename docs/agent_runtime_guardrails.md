# Agent Runtime Guardrails

## 财务基本面完整性硬规则

- A股完整报告、价值报告、长短线合并报告不得在核心财务数据缺失时正常生成。
- 核心财务门槛至少包括：PE/PB、ROE、资产负债率、毛利率、净利率。
- 如果基本面工具返回“工具未返回”“数据不足，不能确认”“无法给出真实PE/PB/PEG”“缺少估值指标”等内容，系统必须先重取东方财富/AKShare等财务数据。
- 如果重取后仍缺核心财务数据，报告状态必须标记为 failed/incomplete，并停止生成价值层和正常结论。
- 不允许用行情数据、技术指标、行业叙事替代财务数据来做长期价值判断。
- 所有后续 000977 浪潮信息及其他A股报告，都必须遵守该规则。

## 执行位置

- `tradingagents/dataflows/fundamentals_quality.py`：统一质量门禁与重取逻辑。
- `tradingagents/agents/utils/agent_utils.py`：基本面工具返回前校验。
- `scripts/run_full_stock_report.py`：完整报告写入前校验。
- `scripts/run_value_layer.py`：价值层生成前校验。

## 用户可见报告禁止截断

- 完整报告、价值报告、长短线合并报告、短线报告和量化报告中，不得出现“原文过长”“内容过长”“已截断”“truncated”等占位内容。
- 如果内容很长，必须写入完整 Markdown 文件；必要时拆分为多个完整附件，而不是在正文中截断。
- 写文件前必须扫描截断标记；一旦发现，报告生成应失败并重跑/拆分。
- 该规则与财务完整性门禁同级，属于 blocker。

## 新闻与基本面分析完整性

- 完整报告必须包含非空新闻分析，不能让 `news_report` 为空后继续生成研究经理、交易员和风险委员会结论。
- 完整报告必须包含正式基本面分析；财务补充数据/三表摘要只是输入，不等同于基本面分析报告。
- 如果新闻或基本面分析段为空、过短或只剩快照，系统必须先后置修复；修复失败则报告标记 failed，不得标记 completed。
- “AKShare补充数据源”只表示原始段未通过完整性门禁后用于补齐数据，不得写成所有上游源都不可用。
